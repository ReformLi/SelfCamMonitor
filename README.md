# 旧手机家用摄像头 —— 最终版需求文档

## 一、项目概述

将旧 Android 手机变为局域网监控摄像头，具备实时画面查看、运动检测报警、自动录像、后台保活等功能。  
**特点**：完全本地运行，无需第三方云服务，免费且开源。

---

## 二、核心功能清单

| 功能            | 状态       | 说明                       |
| ------------- | -------- | ------------------------ |
| 实时视频流 (MJPEG) | ✅        | 通过浏览器查看，仅视频，无音频          |
| 运动检测          | ✅        | 画面变化时自动报警和录像             |
| HTTP 报警推送     | ✅        | 检测到运动时向指定 URL 发送 JSON 数据 |
| 运动触发短视频录制     | ✅        | 检测到运动时自动录制 10 秒 MP4（含音频） |
| 连续录像（按时间分段）   | ❌ 未启用    | 因兼容性问题，暂未开放              |
| 录像存储循环覆盖      | ⚠️ 代码已就绪 | 待连续录像启用后生效               |
| 后台保活          | ✅        | 前台服务、唤醒锁、引导忽略电池优化        |
| 开机自启          | ✅        | 代码支持，部分手机需手动授权           |
| 控制界面          | ✅        | 主界面可监控状态、控制开关            |
| 录像管理          | ✅        | 可浏览、播放、搜索、批量删除录像文件       |

## 三、界面说明与控件作用

### 1. 主界面 (MainActivity)

启动 App 后直接显示的界面，用于控制摄像头服务和查看运行状态。

- **IP 地址** (`tvIpAddress`)  
  显示手机当前 Wi‑Fi 局域网的 IPv4 地址，例如 `192.168.1.5`。  
  浏览器观看视频流时将使用此 IP 地址 + 端口 `8080`。

- **服务状态** (`tvStatus`)  
  显示当前监控服务是否在运行，颜色区分：  
  
  - **绿色 “服务运行中”**：摄像头正在工作，可推流和检测运动。  
  - **红色 “服务已停止”**：服务未启动，无法访问视频流。

- **MJPEG 推流开关** (`switchMjpeg`)  
  打开时：相机画面实时编码为 MJPEG 流，可在浏览器访问 `http://IP:8080/video` 观看。  
  关闭时：视频流停止，浏览器显示提示文字，但相机仍在后台（运动检测仍可能工作）。  
  **实时生效**，无需重启服务。

- **运动检测开关** (`switchMotion`)  
  打开时：持续分析画面变化，检测到运动时自动触发报警和短视频录制。  
  关闭时：不再检测运动，也不报警、不自动录像。推流不受影响。  
  **实时生效**，无需重启服务。

- **录像存储** (`tvStorage`)  
  显示当前所有录像文件占用的总磁盘空间（单位：MB）。  
  录像保存在 `Android/data/com.hpu.selfcammonitor/files/Recordings/`。

- **启动监控** (`btnStart`)  
  点击后：  
  
  1. 检查相机权限（若未授权会弹出权限请求）。  
  2. 请求录音权限（若有声音需求）。  
  3. 引导用户忽略电池优化（若未设置）。  
  4. 启动 `CameraService` 前台服务，通知栏显示“监控运行中”。  
     按钮在服务已启动时禁用。

- **停止监控** (`btnStop`)  
  停止 `CameraService`，释放摄像头，推流和检测全部停止。  
  按钮在服务未启动时禁用。

- **查看录像** (`btnViewRecordings`)  
  跳转到 **录像列表界面**，可浏览、播放、删除已录制的短视频。

---

### 2. 录像列表界面 (RecordingsActivity)

点击主界面“查看录像”后进入，显示所有录制的 MP4 文件。

- **工具栏**
  
  - **搜索按钮**（右上角，普通模式）  
    点击后，搜索栏出现在左侧，包含输入框、清除按钮和“搜索”确认按钮，同时搜索按钮隐藏。  
    输入关键词后点击“搜索”，列表将仅显示文件名包含该关键词的录像。  
    输入框为空时，点击列表空白区域或取消搜索，恢复原始列表。
  
  - **输入框**（搜索模式下显示）  
    输入文件名关键词进行模糊搜索。  
    右侧有 **✕ 清除按钮**，可快速清空输入内容。
  
  - **全选按钮**（多选模式下显示，左上角）  
    点击后选中当前列表中的所有文件，用于批量删除。
  
  - **取消按钮**（多选模式下显示，右上角）  
    退出多选模式，隐藏全选按钮和删除按钮，恢复搜索按钮。

- **录像列表**  
  每一项显示：  
  
  - 文件名（形如 `motion_xxxxxxxxxx.mp4`）  
  - 文件大小（单位 KB）  
    普通模式下短按直接播放；长按进入多选模式。

- **多选模式**  
  
  - 列表中每个项目左侧出现复选框，并自动勾选当前长按的文件。  
  - 再次点击项目可切换选中状态。  
  - 底部出现 **“删除选中 (N)”** 按钮，N 为当前选中的文件数。  
  - 点击删除按钮后，所选文件被永久删除，列表刷新并保留搜索结果（如正在搜索）。  
  - 点击“取消”退出多选，选中状态清空。

- **播放**  
  点击文件会调用系统视频播放器打开，可正常观看录制的有声视频。

## 四、技术实现要点

- **最低 Android 版本**：8.0 (API 26)

- **开发语言**：Kotlin

- **核心依赖**：  
  
  - CameraX（相机控制、图像分析、视频录制）  
  - NanoHTTPD（HTTP 服务器）  
  - OkHttp（报警 HTTP 请求）  
  - AndroidX LocalBroadcastManager（服务与界面通信）

- **视频推流**：使用 CameraX ImageAnalysis 获取 YUV 帧，转为 JPEG，通过 NanoHTTPD 以 MJPEG 格式输出。  

- **运动检测**：相邻帧亮度差异算法，可调节灵敏度（代码已支持）。  

- **短视频录制**：使用 CameraX VideoCapture，录制 10 秒运动触发片段，包含音频（需录音权限）。  

- **后台保活**：前台服务 + PARTIAL_WAKE_LOCK，引导用户关闭电池优化。  

- **存储**：录像保存至应用专属目录，不需存储权限；录像列表使用 FileProvider 供外部播放器打开。

## 五、已知局限与后续优化方向

| 项目           | 现状                     | 建议                 |
| ------------ | ---------------------- | ------------------ |
| 远程音频监听       | 不支持浏览器音频，MJPEG 无音频     | 可升级为 WebRTC 或 RTMP |
| 连续长时间录像      | 未启用（MediaRecorder 兼容性） | 使用 MediaCodec 分段录制 |
| 运动检测灵敏度调节 UI | 底层支持，无界面控件             | 在设置页增加 SeekBar     |
| 报警接收端配置      | 硬编码为空，需用户自行设置          | 在主界面增加 URL 输入框     |
| 开机自启可靠性      | 部分厂商需用户手动授权            | 增加引导说明             |
| 多路观看         | 支持多浏览器同时观看             | 可增加观看人数统计          |
