# 旧手机家用摄像头 —— 最终版需求文档

## 一、项目概述

将旧 Android 手机变为局域网监控摄像头，具备实时画面查看、运动检测报警、自动录像、连续录像、录像管理、后台保活等功能。**特点**：完全本地运行，无需第三方云服务，免费且开源。

---

## 二、核心功能清单

| 功能            | 状态      | 说明                                     |
| ------------- | ------- | -------------------------------------- |
| 实时视频流 (MJPEG) | ✅ 已启用   | 通过浏览器查看，仅视频，无音频                        |
| 运动检测          | ✅ 已启用   | 画面变化时自动报警和短视频录制                        |

## 三、界面说明与控件作用

### 1. 主界面 (`MainActivity`)

启动 App 后直接显示的界面，用于控制摄像头服务和查看运行状态。

- **IP 地址** (`tvIpAddress`)  
  显示手机当前 Wi‑Fi 局域网的 IPv4 地址，例如 `192.168.1.5`。浏览器观看视频流时将使用此 IP 地址 + 端口 `8080`（`http://手机IP:8080/video`），支持 HTTP 基本认证（用户名/密码在设置中配置）。

- **服务状态** (`tvStatus`)  
  显示当前监控服务是否在运行，颜色区分：  
  
  - **绿色 “服务运行中”**：摄像头正在工作，可推流和检测运动。  
  - **红色 “服务已停止”**：服务未启动，无法访问视频流。

- **MJPEG 推流开关** (`switchMjpeg`)  
  打开时：相机画面实时编码为 MJPEG 流，可在浏览器访问。关闭时：视频流停止，但相机仍在后台（运动检测和录像仍可能工作）。**实时生效**。

- **录像模式选择**（两个互斥 `SwitchCompat`）  
  
  - **连续录像**：开启后持续录制，并按设置的分段时长自动分割文件（默认 10 分钟一段）。  
  - **运动触发录像**：开启后仅当检测到运动时录制短视频，时长可在设置中调整（默认 10 秒）。  
  - 两个开关不能同时打开，均不选时表示“仅预览”（不录像、不运动检测）。**实时生效**。

- **录像存储** (`tvStorage`)  
  显示当前所有录像文件占用的总磁盘空间（单位：MB），递归计算所有日期子文件夹内的 MP4 文件。每次回到主界面自动刷新。

- **启动监控** (`btnStart`)  
  点击后依次检查相机权限、录音权限、引导忽略电池优化，然后启动 `CameraService` 前台服务，通知栏显示“监控运行中”。

- **停止监控** (`btnStop`)  
  停止 `CameraService`，释放摄像头，推流和检测全部停止。

- **查看录像** (`btnViewRecordings`)  
  跳转到**录像列表界面**，按日期文件夹展示录像。

- **设置按钮**（右上角齿轮图标）  
  跳转到 `SettingsActivity`，配置详细参数（分辨率、帧率、灵敏度、报警 URL、录像时长等）。

---

### 2. 录像列表界面 (`RecordingsActivity`)

点击主界面“查看录像”后进入，按日期文件夹（格式 `yyyy-MM-dd`）组织所有录像，每个文件夹显示包含的视频数量和总大小。

- **普通模式**  
  
  - 顶部中央显示文件夹个数。  
  - 点击任意文件夹进入该文件夹的视频列表页面 (`VideoListActivity`)。  
  - 长按文件夹进入**多选模式**。

- **多选模式**  
  
  - 顶部改为：左侧“全选”按钮，中央“已选 X 项”，右侧“取消”按钮。  
  - 每个文件夹左侧出现复选框，可勾选多个文件夹。  
  - 底部显示“删除选中”和“导出选中”按钮。  
  - 点击“删除”会弹窗确认，递归删除文件夹内所有视频。  
  - 点击“导出”通过系统文档树选择目标目录，递归复制文件夹及内容。  
  - 按返回键退出多选模式。

---

### 3. 视频列表界面 (`VideoListActivity`)

展示单个日期文件夹内的所有 MP4 文件。

- **顶部工具栏**  
  
  - 普通模式：左侧返回按钮，中央显示文件夹名，右侧显示文件总数。  
  - 多选模式：左侧“全选”按钮，中央“已选 X 项”，右侧“取消”按钮，同时隐藏返回按钮和文件计数。

- **搜索栏**（始终显示）  
  
  - 输入框 + 清除按钮（X） + 搜索按钮。  
  - 输入文字时 X 出现，点击清空输入框并立即恢复全部视频列表（自动搜索空关键词）。  
  - 输入框为空时点击 X 则隐藏键盘并失去焦点。  
  - 点击搜索按钮或按键盘搜索键执行过滤（文件名包含关键词）。

- **视频列表**  
  每一项显示：文件名、文件大小（KB）、录制日期时间（从文件名解析或取文件修改时间）、视频时长（mm:ss 或 HH:mm:ss）。  
  
  - 普通模式：短按播放视频，长按进入多选模式。  
  - 多选模式：每个视频左侧出现复选框，顶部计数实时更新，底部显示“删除选中”和“导出选中”。  
  - 删除视频时弹窗确认，支持批量删除。  
  - 导出功能：选择外部文件夹后，将选中的视频文件复制到目标位置（支持 MediaStore 刷新）。  
  - 按返回键退出多选模式。

- **元数据异步加载**  
  首次进入或文件列表变化时，后台线程读取所有视频的时长（`MediaMetadataRetriever`）和日期时间，加载完成后刷新列表，避免 UI 阻塞。同时使用内存缓存避免重复解析。

### 4. 设置界面 (`SettingsActivity`)

- **视频设置**  
  
  - 推流分辨率（动态读取摄像头支持的分辨率列表，例如 640x480、1280x720 等）。  
  - 推流帧率（1～16 fps，对应 CameraX 实际上限）。

- **运动检测设置**  
  
  - 动作识别灵敏度（0～100，数值越低越敏感）。  
  - 报警静默期（秒），避免频繁发送报警。  
  - 报警接收 URL（支持 HTTP GET 请求，携带运动时间戳）。

- **录像设置**  
  
  - 录像模式（连续/运动触发/仅预览）—— **与主界面开关同步**。  
  - 运动录像时长（10秒、30秒、1分钟、3分钟、5分钟）。  
  - 连续录像分段时长（1分钟、5分钟、10分钟、30分钟、1小时）。  
  - 注意：修改分段时长不会中断当前正在录制的片段，新设置对下一个片段生效。

- **系统设置**  
  
  - 开机自启开关（保存到 SharedPreferences，由 `BootReceiver` 实现）。  
  - 监控时间限制（可设置每日起止时间，非监控时段相机不分析也不录像）。

- **安全设置**  
  
  - HTTP 认证用户名/密码（基本认证，保护 MJPEG 流）。

> 所有设置（除分辨率、帧率需重启监控外）均实时生效，通过 `RELOAD_CONFIG` 广播通知 `CameraService` 更新行为。

## 四、技术实现要点

- **最低 Android 版本**：8.0 (API 26)  

- **开发语言**：Kotlin  

- **核心依赖**：  
  
  - CameraX（相机控制、图像分析、视频录制）  
  - NanoHTTPD（HTTP 服务器，提供 MJPEG 流）  
  - OkHttp（报警 HTTP 请求）  
  - AndroidX LocalBroadcastManager（服务与界面通信）  

- **视频推流**：  
  
  - CameraX `ImageAnalysis` 获取 YUV 帧，转为 NV21，再压缩为 JPEG，通过 NanoHTTPD 以 MJPEG 格式输出。  
  - 根据 `ImageProxy` 的旋转角度（`imageInfo.rotationDegrees`）动态旋转 JPEG，保证输出画面与录像方向一致，消除红色虚影。  
  - 使用 `Bitmap` 旋转方案（解码→旋转→编码），保证颜色正确，性能可接受（640x480）。

- **运动检测**：  
  相邻帧亮度差异算法，支持灵敏度调节。

- **录像存储**：  
  
  - 录像文件按日期存放到 `Recordings/yyyy-MM-dd/` 子文件夹。  
  - 连续录像：在 `CameraService` 中维护一个 `Recording` 会话，通过 `Handler` 定时停止当前分段并自动开始下一段。  
  - 运动触发录像：单次录制指定时长（从设置读取）。  
  - 文件命名规则：`motion_YYMMDDHHmmss_xxx.mp4`（运动触发）或 `continuous_<timestamp>.mp4`（连续录像）。

- **录像管理**：  
  
  - `RecordingsActivity` 扫描 `Recordings` 下所有日期文件夹，统计视频数量和总大小。  
  - `VideoListActivity` 递归计算文件夹内 MP4 文件，异步加载时长和日期，支持搜索、多选、删除、导出。  
  - 导出使用 `DocumentsContract` / `MediaStore` API，兼容 Android 10+ 分区存储。

- **后台保活**：  
  前台服务 + `PARTIAL_WAKE_LOCK`，引导用户关闭电池优化。

- **横屏统一**：  
  强制 `ImageAnalysis` 和 `VideoCapture` 的 `targetRotation = Surface.ROTATION_0`，确保输出画面宽>高，与录像方向一致。
