# 旧手机家用摄像头项目规划

## 一、项目概述

将旧 Android 手机变为局域网监控摄像头，具备实时画面查看、运动检测报警、自动录像、后台保活等功能。  
**特点**：完全本地运行，无需第三方云服务，免费且开源。

---

## 二、已实现核心功能

| 功能            | 说明                                            |
| ------------- | --------------------------------------------- |
| 实时视频流 (MJPEG) | 浏览器通过 `http://IP:8080/video` 观看，可设置 HTTP 基本认证 |
| 运动检测          | 画面变化时自动触发报警和录像，灵敏度可调                          |
| HTTP 报警推送     | 检测到运动时向用户配置的 URL 发送 JSON 数据                   |
| 运动触发短视频录制     | 检测到运动时自动录制 10 秒 MP4（含音频）                      |
| 控制界面          | 主界面显示 IP、状态，可开关 MJPEG、运动检测；查看录像列表             |
| 录像管理          | 播放、搜索、多选删除录像文件                                |
| 后台保活          | 前台服务、唤醒锁、引导忽略电池优化                             |
| 开机自启          | 代码支持，需用户授权（设置中可开关）                            |
| 用户可配置参数       | 分辨率、帧率、灵敏度、报警 URL/静默期、监控时段、HTTP 认证            |

---

## 三、设置页面参数说明

| 参数            | 类型   | 默认值         | 说明                              |
| ------------- | ---- | ----------- | ------------------------------- |
| 推流分辨率         | 下拉选择 | 640x480     | 可选 320x240~1920x1080，影响画面清晰度及带宽 |
| 推流帧率          | 滑动条  | 10 fps      | 软限制，实际帧率受设备性能影响                 |
| 运动检测灵敏度       | 滑动条  | 50          | 数值越小越敏感（0-100）                  |
| 开机自启          | 开关   | 关闭          | 控制是否在开机时自动启动服务                  |
| 报警接收 URL      | 文本输入 | 空           | 运动报警目标地址，留空则不发送                 |
| 报警静默期         | 数字输入 | 30 秒        | 同一事件在静默期内不重复报警                  |
| 监控时间限制        | 时间选择 | 00:00~23:59 | 可设定服务只在特定时段运行（待启用）              |
| HTTP 认证用户名/密码 | 文本输入 | 空           | 设置后浏览器需凭据才能观看视频流                |

## 五、界面结构

### 1. 主界面

- IP 地址显示
- 服务状态指示灯
- MJPEG 推流开关
- 运动检测开关
- 录像存储占用
- 启动/停止监控按钮
- 查看录像按钮
- 设置按钮（进入 SettingsActivity）

### 2. 设置界面

- 推流分辨率（Spinner）
- 推流帧率（SeekBar + 数值）
- 运动灵敏度（SeekBar + 数值）
- 开机自启（Switch）
- 报警 URL（EditText）
- 报警静默期（EditText）
- 监控时间段（开始/结束时间选择）
- HTTP 认证用户名/密码（EditText）
- 保存 / 取消按钮

### 3. 录像列表界面

- 搜索框（模糊搜索）
- 多选模式（全选、取消、删除选中）
- 点击播放

## 六、技术栈

- 语言：Kotlin
- 最低 Android 版本：8.0 (API 26)
- 核心依赖：CameraX, NanoHTTPD, OkHttp, LocalBroadcastManager
- 推流：CameraX ImageAnalysis → JPEG → MJPEG over HTTP
- 录像：CameraX VideoCapture (运动触发)
- 报警：HTTP POST JSON to user-defined URL
- 存储：应用专属目录 (`Android/data/.../files/Recordings`)
